libDatabase

libDatabase is a database abstraction library written in Transcript AND built ON top of RevDB AND the Valentina externals. It facilitates working with database record sets, adding AND updating records AND import/exporting SQL dumps, AND exporting from one database vendor to another.

command libdb_exportDB (pSourceDB, pOutputDB, pOutputFile, pTables, pSQL, pExportTables, pExportData, pTranslateA)
Description:
Exports a database to either a) another database OR b) a file. IF exporting to another database THEN you can either import data directly into it OR just create a dump file (provide pOutputFile) that can be inserted into the database at a later time. IF the output database type is Valentina OR SQLite AND not output file is provided THEN the database will be created IF it doesn't exist. The connection to the output database will be closed when this handler finishes. used for Valentina IF you need to add methods, etc. Each statement should END with a ";" AND have a cr following it.
Parameters:
pSourceDB: Source database identifier.
pOutputDB: Output database identifier. Leave empty AND provide an output file to get a dump of pSourceDB.
pOutputFile: Path to file to export to. Leave empty to transfer data directly from one db to another.
pTables: List of tables to export. Leave empty to export entire db.
pSQL: Additional SQL statements to execute after creating database. This can be
pExportTables: set to false to skip creating table commands when exporting. DEFAULT is true.
pExportData: set to false to skip inserting data when exporting. DEFAULT is true.
pTranslateA: A translation array. NOT DOCUMENTED.
Return:
Empty
Throws:
libdberr

function libdb_getCursorRecords (pCursorA, @pDataA, pKey)
Description:
Populates pDataA with all records from the provided cursor. When using the DEFAULT pKey of "Data", the array contains a "Length" key which contains the number of records in the array. IF you provide a value for pKey that is different than pKey THEN the lenght is found in [pKey,Length]. Data in the array is accessed by [pKey,INDEX,COLUMNNAME].

Note that unlike libdb_getTableRecords, libdb_getCursorRecords does not overwrite any data currently in the pDataA variable. This can be used for compounding results into one container by changing the value of pKey.

Parameters:
pCursorA: cursor array.
@pDataA: Array that will hold returned records. Passed by reference.
pKey: By DEFAULT array is indexed using "Data",INDEX,FIELD. Use pKey to replace the value of "Data".
Return:
Empty
Throws:
libdberr

function libdb_getTableRecords (pDBKey, @pDataA, pSQL, pEncodingsA)
Description:
Populates pDataA with the returned rows form the SQL query. The number of rows is stored in pDataA["Length"]. The row data is stored in pDataA["Data",i,"ColumnName"] where i represents the record index whose values you want. pDataA["Length"] is equal to 0 IF no records were found. pDataA also has a key named "ColumnNames" which is a comma delimited list of all columns from the SQL query. For example, IF your query was "SELECT ID, Name FROM MyTable" THEN pDataA["Columns"] = "ID,Name".
Parameters:
pDBKey: DB to get data from.
@pDataA: Array that will hold returned records. Passed by reference.
pSQL: SQL query to execute.
pEncodingsA: Array describing encodings for columns in results.
Return:
Empty
Throws:
libdberr

function libdb_makeSearchString (pDBType, pTerm, pType, pCaseSensitive, pUseRegex)
Description:
Returns a modified version of pTerm that can be used in a LIKE query. For Valentina 1.x this looks like '\ApTerm', 'pTerm\Z', '\ApTerm\Z' OR 'pTerm'. For all other databases the returned value is '%pTerm', 'pTerm%' OR '%pTerm%'.
Parameters:
pDBType: Database type (mysql, odbc, oracle, postgresql, sqlite, valentina, valentina2).
pTerm: Term to search on.
pType: BEGINS, ENDS, CONTAINS, EXACT. DEFAULT is CONTAINS. EXACT is only useful for Valentina 1.x databases.
pCaseSensitive: Whether search string is CASE sensitive OR not (Valentina only).
pUseRegex: Whether to escape regex characters (Valentina only).
Return:
String

function libdb_openCursor (pDBKey, pSQL, pEncodingsA, pDontCache, pLocation, pLockType, pDirection)
Description:
This handler will execute the provided sql AND return cursor array. A cursor array has the following keys:

[ID,Cursor,DBKey,Encodings...]

The cursor element is the database cursor that can be used with RevDB OR Valentina calls. The other keys are used internally by libDB.

Parameters:
pDBKey: target database.
pSQL: SQL to execute.
pEncodingsA: Array describing encodings for columns in results.
pDontCache: pass true IF you will be working with a cursor that will contain large amounts of data (i.e. binary) that shouldn't be cached.
pLocation: Valentina only. "client" OR "server". Valentina 2: DEFAULT is "server" for local connection, "client" for client/server. Valentina 1.x: DEFAULT is "server".
pLockType: Valentina only. "none", "read" OR "readwrite". DEFAULT is "none".
pDirection: Valentina only. "forward" OR "random". DEFAULT is "random".
Return:
cursor array
Throws:
libdberr

command libdb_registerDatabase (pDBKey, pSettingsA)
Description:
Registers the connection parameters for a database you plan to connect to. The type parameter can be "valentina", "mysql" OR "sqlite". All databases can have a "Version" key which specifies the version of the database being connected to. keys for pSettingsA array -
  • Valentina - Type, Version, Name, CacheSize, MacSerial, WinSerial, EncryptionKey, EncryptionType, DebugLevel, FlushCount, Mode, SegmentSize, Host, Username, Password, Port, Timeout, OSEncoding, StorageType
  • SQLite - Type="sqlite", FilePath, Email, SerialNum, version (2,3)
  • Other Databases - Type, Host, Name, Username, Password, UseSSL, version
Parameters:
pDBKey: String reference that will be used to designate this database connection.
pSettingsA: An array whose keys correspond to database connection settings.
Return:
empty

function libdb_updateTable (pDBKey, pTable, pColumnValuesA, pSearchA, pEncodingsA)
Description:
Updates columns in a table based ON values in pDataA array.
Parameters:
pDBKey: target DB.
pTable: target table.
pColumnValuesA: An array of data whose keys correspond to the column names you want to update. Any string values will be escaped for you. Provide a comma separated list of IDs in one of the pSearchFields keys to update multiple records with the same information.
